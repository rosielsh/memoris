# 아이디어

**"정리는 완벽한데 복습은 안 하는"** 학습자들을 위한 **자동 복습 시스템**

웹 링크나 pdf를 포함한 학습 자료를 한 번 연결하면, AI가 자동으로 문제를 생성하고 원하는 타이밍에 복습 알림을 보내주는 서비스

<br/>

# 해결하고자 하는 문제

### 현재 상황

- 노션, 구글 독스 등에 학습 내용을 열심히 정리하고 정리 후에는 거의 다시 보지 않음
- 따라서 시험이나 필요할 때 다시 처음부터 공부해야 함
- **즉, "정리만 하고 기억은 못 하는"** 비효율적 학습

## 사용자 스토리와 인수 조건

### 예시 1: AI 문제 자동 생성 기능

**요구사항**: AI 문제 자동 생성 기능

**사용자 스토리**: "대학생으로서, 저는 정리한 학습 자료에서 자동으로 문제가 생성되기를 원합니다. 그래서 복습할 때 능동적으로 학습할 수 있습니다."

**인수 조건**:

- Given: 사용자가 학습 자료(PDF, 웹링크, 노션 등)를 시스템에 연결했을 때
- When: 복습 시간이 되어 문제 생성을 요청했을 때
- Then: 해당 자료의 핵심 내용을 바탕으로 5-10개의 문제가 자동 생성된다

### 예시 2: 면접 예상 질문 생성 기능

**요구사항**: 면접 예상 질문 형태 문제 생성 기능

**사용자 스토리**: "개발자 취업 준비생으로서, 저는 정리한 CS 지식이 실제 면접에서 어떻게 질문될지 미리 연습하고 싶습니다. 그래서 면접 상황에 자신감을 가질 수 있습니다."

**인수 조건**:

- Given: 사용자가 CS 관련 학습 자료를 시스템에 연결했을 때
- When: 면접 대비 모드로 문제 생성을 요청했을 때
- Then: "OOP의 4가지 특징을 설명해보세요", "데드락이 무엇인지 설명하고 해결 방법을 제시해보세요" 등 실제 면접에서 나올 법한 질문 형태로 문제가 생성된다

### 예시 3: 학습 취약점 분석 기능

**요구사항**: 학습 취약점 분석 및 맞춤 스케줄링 기능

**사용자 스토리**: "직장인으로서, 저는 제가 어떤 부분을 잘 기억하지 못하는지 파악하고 싶습니다. 그래서 약한 부분을 더 집중적으로 복습할 수 있습니다."

**인수 조건**:

- Given: 사용자가 여러 차례 복습 문제를 풀고 결과 데이터가 축적되었을 때
- When: 사용자가 학습 분석 리포트를 요청했을 때
- Then: 정답률이 낮은 주제나 개념을 식별하고, 해당 부분의 복습 빈도를 자동으로 증가시킨다

## 💡 핵심 가치 제안

1. **학습 효율성 극대화**: 정리만 하고 끝나는 비효율을 해결하여 진짜 '기억하는' 학습으로 전환
2. **자동화된 복습 관리**: 사용자가 신경 쓰지 않아도 최적의 타이밍에 복습 기회 제공
3. **개인화된 학습 경험**: AI가 개인의 학습 패턴과 취약점을 분석하여 맞춤형 복습 제공
4. **지속가능한 학습 습관**: 부담 없는 알림과 짧은 복습으로 장기적인 학습 습관 형성

## 🎯 핵심 기능

- **스마트 학습 자료 연동**: 웹링크, PDF, 노션 등 다양한 형태의 학습 자료 자동 분석
- **AI 기반 문제 생성**: 학습 내용의 핵심을 파악하여 다양한 유형의 문제 자동 생성
- **적응형 복습 스케줄링**: 망각 곡선과 개인 학습 패턴을 고려한 최적 복습 타이밍 제공
- **학습 성과 추적**: 복습 효과를 시각화하고 학습 성장을 측정할 수 있는 대시보드

## 🚀 구현 범위 (MVP)

- **기본 학습 자료 연동**: PDF 업로드 및 웹링크 연결 기능
- **간단한 문제 생성**: 객관식, 주관식 기본 문제 생성 알고리즘
- **복습 알림 시스템**: 이메일/앱 푸시 기반 정기 알림 기능
- **기본 학습 추적**: 문제 정답률 및 복습 빈도 기록 기능
